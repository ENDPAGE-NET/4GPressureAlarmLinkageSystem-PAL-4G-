# WH-GM5 设备对接指南

本文档说明如何将客户现场的 WH-GM5 (LTE Cat-1) 4G 模块从有人云切换到自建 EMQX 服务器。

## 1. 硬件信息

| 项目 | 详情 |
|------|------|
| 模块型号 | WH-GM5TF-CT-4-N40 (有人物联网 usr.cn) |
| 通信方式 | LTE Cat-1 (4G) |
| MQTT 版本 | 3.1.1 |
| 串口参数 | 115200, 8N1, 无校验 |
| 配置工具 | 4G CAT1 setup V1.4.2 |

## 2. 对接流程总览

```
管理员在 Web 面板创建设备
        ↓
系统自动生成 MQTT 凭证
        ↓
管理员把凭证发给客户
        ↓
客户用 4G CAT1 setup 软件配置设备
        ↓
设备通过 4G 连接到 EMQX Broker
        ↓
后端接收设备数据，完成业务闭环
```

## 3. 在 Web 面板创建设备

1. 管理员登录 Web 后台
2. 进入「设备管理」页面，点击「创建设备」
3. 输入设备名称和序列号 (SN)
4. 创建成功后，系统自动弹出 **MQTT 接入配置卡片**，包含：
   - 服务器地址和端口
   - MQTT 用户名（默认为设备 SN）
   - MQTT 密码（系统自动生成）
   - 客户端 ID
   - 发布 Topic（上行）
   - 订阅 Topic（下行）
5. 点击「复制配置」将信息发送给客户

## 4. 客户端配置设备

客户需使用 **4G CAT1 setup V1.4.2** 配置工具，通过串口连接设备进行配置。

### 4.1 基本步骤

1. 用 USB 转串口线连接设备和电脑
2. 打开 4G CAT1 setup 软件，选择正确的 COM 端口
3. 选择产品型号：`WH-GM5TF-X-N40-4`
4. 选择工作模式：**MQTT 模式**
5. 选择 **通用 MQTT**（不选阿里云）
6. 填写 MQTT 参数（从 Web 面板复制的配置）：

| 配置项 | 填入内容 |
|--------|----------|
| 服务器地址和端口 | `usr-iot.endpage.net` 端口 `1883`（或 `8883` for TLS） |
| 用户名 | Web 面板提供的 mqtt_username |
| 密码 | Web 面板提供的 mqtt_password |
| 客户端 ID | Web 面板提供的 mqtt_client_id |
| MQTT 版本 | `3.1.1` |

7. 点击写入/保存配置
8. 设备会自动重启并连接到新的 MQTT 服务器

### 4.2 通过 AT 指令配置（可选）

如果无法使用配置软件，可通过串口发送 AT 指令：

```
+++              # 进入配置模式（等待返回 'a'）
a                # 确认进入（等待返回 '+ok'）
AT+WKMOD=CMD     # 切换到 CMD 模式
AT+S             # 保存并重启

# 重启后执行驻网和 MQTT 连接
AT+CPIN?                    # 检查 SIM 卡
AT+CSQ?                     # 检查信号
AT+CGATT=1                  # 网络附着
AT+CGACT=1,1                # 激活网络

# MQTT 连接
AT+MQTTCONN="usr-iot.endpage.net",1883,"客户端ID",120,1,"用户名","密码"
AT+MQTTSUBUNSUB="pal4g/devices/{SN}/down",1,1     # 订阅下行
AT+MQTTPUB="pal4g/devices/{SN}/up","test",1,0,0    # 测试发布
AT+MQTTSTA                                          # 检查连接状态
```

## 5. Topic 结构

| 方向 | Topic 格式 | 说明 |
|------|-----------|------|
| 上行（设备→服务器） | `pal4g/devices/{SN}/up` | 设备状态上报、报警数据 |
| 下行（服务器→设备） | `pal4g/devices/{SN}/down` | 继电器控制指令 |

## 6. 数据协议（待确认）

### 6.1 协议发现阶段

由于 MCU 固件的串口数据协议尚未完全确认，系统已内置**原始数据捕获**功能：

- 设备接入后，所有上行数据都会被完整记录到通信日志
- 在 Web 后台「日志中心」查看 `mqtt_transparent_raw` 类型的日志
- 日志中包含原始文本和十六进制表示

### 6.2 推荐 JSON 协议（如可修改 MCU 固件）

## 7. 验证设备连接

1. 在 Web 后台「MQTT 状态」页面检查 Broker 连接状态
2. 设备配置完成后，在「设备管理」页面查看设备是否上线
3. 在「日志中心」查看通信日志，确认收到设备数据
4. 如果设备发送的是原始格式，在日志中分析数据结构后可适配解析器

## 8. EMQX 服务端配置

### 8.1 HTTP 认证

在 EMQX Dashboard -> Authentication 中添加 HTTP Server 认证器：

- **Method:** POST
- **URL:** `http://127.0.0.1:8001/api/v1/mqtt/emqx/auth`
- **Body:** `{"username":"${username}","password":"${password}","clientid":"${clientid}"}`
- **Headers:** `Content-Type: application/json`

### 8.2 ACL 鉴权（可选）

在 EMQX Dashboard -> Authorization 中添加 HTTP Server 授权器：

- **Method:** POST
- **URL:** `http://127.0.0.1:8001/api/v1/mqtt/emqx/acl`
- **Body:** `{"username":"${username}","clientid":"${clientid}","topic":"${topic}","action":"${action}"}`

## 9. 常见问题

**Q: 设备连不上 MQTT 服务器？**
- 检查服务器地址和端口是否正确
- 检查防火墙是否开放 1883/8883 端口
- 检查 SIM 卡是否有流量、信号是否正常（AT+CSQ?）
- 检查 EMQX 认证配置是否正确

**Q: 设备连上了但后台看不到数据？**
- 检查设备的发布 Topic 是否与后台订阅的 Topic 匹配
- 在 EMQX Dashboard 中查看设备在线状态和消息流
- 检查后台通信日志是否有收到消息

**Q: 多个模块(A-F)如何管理？**
- 每个物理模块有独立的 WH-GM5 和 SIM 卡，各自独立接入
- 在 Web 面板中为每个模块分别创建设备，或在设备下添加子模块
- 联动组（DeviceGroup）用于管理同一套设备的报警联动关系
